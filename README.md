# Мини QA-проект: API тесты на `pytest`

Небольшой, но практичный проект для QA-инженера: автоматизированные API-проверки с локальным mock-сервером.

## Что внутри

- `pytest`
- API-клиент на стандартной библиотеке Python (без внешних зависимостей для HTTP)
- локальный mock REST API для офлайн-запуска
- позитивные и негативные проверки
- параметризация тестов

## Структура

```text
.
├── pytest.ini
├── requirements.txt
├── src
│   ├── client.py
│   └── mock_server.py
└── tests
    ├── conftest.py
    └── test_posts_api.py
```

## Быстрый старт

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pytest -v
```

## Какие проверки реализованы

- `GET /posts/1` → `200` + проверка схемы ответа
- параметризованные проверки нескольких постов (`1`, `10`, `100`)
- `GET /posts/999999` → `404`
- `POST /posts` → `201` + проверка, что payload возвращается с `id`

## Почему проект «дельный»

- Покрывает ключевые QA-паттерны API-тестирования
- Работает полностью локально (удобно для обучения, демо и CI)
- Легко расширяется новыми эндпоинтами, валидациями и слоями отчётности
